# ATS Resume Optimizer

Transform any resume into one that parses cleanly through modern ATS (including AI semantic parsers) and ranks highly by demonstrating strong, evidenced fit for a target role.

## When to Use
Activate on any request involving resume improvement for job applications, ATS compliance, keyword optimization against a JD, before-and-after rewrites, or creating tailored versions. Always request the target job description (JD) and current resume (text, PDF, or DOCX) if not provided.

## Core Workflow (Follow Step-by-Step)

1. **Gather Inputs and Analyze JD**
   - Extract must-have vs preferred requirements, exact keywords/phrases, tools, responsibilities, qualifications, and soft skills.
   - Note pain points the role aims to solve and company context if available.
   - Output: Prioritized keyword list and responsibility map.

2. **Audit Current Resume for ATS Compatibility**
   - Use `references/ats-audit-checklist.md` as primary checklist.
   - Identify formatting issues (multi-column, tables, graphics, headers/footers, non-standard fonts, image-based PDF).
   - Flag content gaps: missing keywords, weak/non-quantified bullets, generic summary, irrelevant sections bloating length.
   - Confirm contact info is clean and prominent at top.
   - Decide on fixes: Restructure to single-column if needed; plan content cuts or expansions.

3. **Map and Integrate Keywords Semantically**
   - Cross-reference resume content against JD keyword list.
   - Weave exact or natural variations into Professional Summary, Skills section, and experience bullets *in context of achievements*.
   - Prioritize hard skills and JD-specific terminology. Include synonyms where they fit naturally.
   - Target 70-90% coverage of critical terms without stuffing. Modern semantic ATS reward contextual understanding and proof over raw count.
   - Reference `references/ats-best-practices.md` for 2026 semantic matching guidance.

4. **Strengthen and Quantify All Achievements**
   - Replace weak verbs with strong ones from `references/action-verbs.md`.
   - Convert every bullet to: **Action Verb + Task/Scope + Result (with metric or clear impact)**.
   - Add or estimate numbers: %, revenue, time saved, users impacted, scale, rankings, error reduction, efficiency gains.
   - Align bullets to JD responsibilities (show you have solved similar problems).
   - Show career progression and increasing scope where possible.
   - Limit to 4-6 strong bullets for recent/relevant roles; condense older ones to 1-3.

5. **Rewrite Key Sections for Maximum Impact**
   - **Professional Summary**: 3-5 lines. Lead with experience level + target title + top 2-3 matching expertise areas + one signature quantified win + forward-looking value to this role/company.
   - **Skills Section**: Make prominent and scannable. List or categorize with JD matches first. Include Technical Skills, Tools/Platforms, Methodologies, and only evidenced soft skills.
   - **Experience**: Reorder bullets or roles if needed so strongest JD alignment appears earliest. Use consistent "Month Year – Present" dates.
   - **Education/Certs**: Clean, exact names, dates, and relevant details only. Add projects or coursework sparingly.
   - **Optional Sections**: Projects or Volunteer only if they inject critical keywords or demonstrate relevant passion/impact. Otherwise remove to tighten length.

6. **Apply ATS-Safe Formatting**
   - Enforce single-column, standard headings, clean fonts, standard bullets, consistent styling.
   - Ensure all text is extractable (test copy-paste to plain text).
   - Recommend final export as .docx (best compatibility) or clean text PDF.
   - If user requests polished file output, coordinate with pdf or docx skill after text optimization.

7. **Validate and Score**
   - Re-run full `references/ats-audit-checklist.md`.
   - Estimate keyword coverage and semantic fit.
   - Suggest user tests with free tools like Jobscan, Teal, or Resume Worded (target 80%+ match on must-haves).
   - Provide before/after examples for major changes (especially Summary and 2-3 key bullets).
   - Deliver full rewritten resume in clean markdown + specific edit list, or step-by-step implementation guide.

8. **Exceed Expectations (Differentiation Tactics)**
   - Ensure the resume tells a compelling, evidence-backed story of fit within the first half-page.
   - Use metrics and business impact language that mirrors JD priorities.
   - For technical/AI roles: Specify scale, tools versions, methodologies (e.g., "fine-tuned LLMs", "built RAG pipelines", "deployed agents reducing X by Y%").
   - Demonstrate not just "can do the job" but "have done it successfully with measurable results".
   - Keep human skim in mind: Key proof points must jump out in 6-10 seconds.

## Output Formats
- Detailed optimization report with keyword gap analysis, specific rewrite suggestions, and checklist scores.
- Full rewritten resume in clean, ATS-ready markdown (ready to paste into Word or use with docx/pdf skill).
- Before-and-after comparisons for critical sections.
- Final recommendations on file format, additional tailoring per application, and submission tips.

- Always prioritize honesty in metrics and claims while maximizing persuasive impact.

This workflow produces resumes that not only pass automated screens but position candidates as the clearest, most evidenced match for the role.
